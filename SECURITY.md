# Security Policy

## Supported Versions

This is a learning portfolio. Only the latest commit on `main` is supported.

## Reporting a Vulnerability

If you discover a security vulnerability in any project in this repository:

1. **Do NOT open a public GitHub issue.**
2. Open a [GitHub Security Advisory](https://github.com/YOUR_USERNAME/cybersecurity-projects/security/advisories/new) (private disclosure).
3. Include: affected project number, description of the vulnerability, reproduction steps, and potential impact.

You will receive a response within **72 hours**. If the issue is confirmed, a fix will be released as soon as possible.

## Scope

- Vulnerabilities in project source code (e.g., command injection, insecure crypto usage, path traversal)
- Vulnerable dependencies (also report via Dependabot or a security advisory)

## Out of Scope

- Vulnerabilities in Kali Linux itself or its tools
- Theoretical vulnerabilities without a proof of concept
- Issues in example output or documentation

## Content Policy

This repository contains only defensive (blue-team) tools. If you believe any project crosses into offensive territory, please report it via a Security Advisory — this is treated with the same urgency as a code vulnerability.
